1. Core Data Concepts

1.1 Types of Data

  • Structured: Fixed schema, rows & columns, easy SQL queries.
  • Semi-Structured: Flexible schema (JSON, XML, Avro, Parquet, ORC).
  • Unstructured: No schema (images, video, PDFs, social posts).

1.2 Common Data File Formats

Format Type Key Facts
CSV Structured Plain text, comma separated, universal but no data types
JSON Semi-Structured Human-readable, key-value pairs, nested objects allowed
XML Semi-Structured Tag-based, verbose, widely used in older systems
Avro Semi-Structured Row-based binary, schema in header, great for streaming
Parquet Structured Column-based binary, excellent for analytics queries
ORC Structured Column-based, optimized for Hive/big data workloads

1.3 Types of Databases

  • Relational: Tables, SQL, ACID compliance, transactional workloads.
  • Non-Relational: Key-Value, Document, Column-family, Graph.

1.4 Data Workloads

Characteristic OLTP OLAP
Operations Insert/update/delete Complex reads
Data per query Small Large
Schema Normalized Denormalized
Response Time Milliseconds Seconds to minutes
Users Many concurrent Fewer, analysts

1.5 Data Roles

  • Database Administrator: Manages, secures, tunes databases.
  • Data Engineer: Builds pipelines, moves/transforms data.
  • Data Analyst: Queries, reports, dashboards, insights.

2. Relational Data in Azure

2.1 Relational Concepts

  • Table, Row, Column
  • Primary Key, Foreign Key
  • Index, View, Stored Procedure
  • Normalization: Reduce duplication, improve integrity (tradeoff: more joins).

2.2 SQL Basics

-- DML
SELECT CustomerName, Email FROM Customers WHERE Country = 'UK';
INSERT INTO Orders (CustomerID, Amount) VALUES (1, 299.99);
UPDATE Orders SET Amount = 350 WHERE OrderID = 42;
DELETE FROM Orders WHERE OrderID = 42;

-- DDL
CREATE TABLE Products (ProductID INT PRIMARY KEY, ProductName VARCHAR(100));
ALTER TABLE Products ADD Price DECIMAL(10,2);
DROP TABLE Products;

3. Non-Relational Data on Azure

3.1 Azure Storage

  • Blob Storage: Block blobs (files), Append blobs (logs), Page blobs (VHDs). Tiers: Hot, Cool, Archive.
  • File Storage: SMB file shares, lift-and-shift apps.
  • Table Storage: Key-value NoSQL, PartitionKey + RowKey, scalable and low cost.

3.2 Azure Cosmos DB

  • Globally distributed, multi-model NoSQL.
  • Consistency levels: Strong → Eventual.
  • Multi-master writes, low latency, elastic scalability.

Cosmos DB APIs

API Data Model Use Case

Azure Services Data Factory, Synapse Stream Analytics, Event Hubs

4.2 Azure Data Services for Analytics

  • Azure Data Factory: Cloud ETL/ELT, 90+ connectors, orchestrator.
  • Azure Data Lake Storage Gen2: Raw data landing zone, hierarchical namespace, ACL security.
  • Azure Synapse Analytics: SQL pools, Spark, pipelines, Cosmos DB link — central analytics hub.
  • Azure Databricks: Apache Spark notebooks, advanced ML & engineering.
  • Microsoft Fabric: Unified SaaS analytics (OneLake, Power BI, Synapse merged).
  • Azure HDInsight: Managed Hadoop/Spark/Hive/Kafka — best for open-source teams.

4.3 Real-Time Analytics

  • Azure Event Hubs: Real-time ingestion, millions of events/sec, Kafka-compatible.
  • Azure Stream Analytics: SQL-like queries for streams, reads from Event Hubs/IoT Hub, outputs to SQL, Cosmos, Power BI, Blob.
  • Azure IoT Hub: Two-way device-cloud communication, device management.

4.4 Power BI

Microsoft’s business intelligence and visualization tool.

  • Power BI Desktop: Free Windows app for building reports.
  • Power BI Service: Cloud platform for publishing/sharing.
  • Power BI Mobile: iOS/Android apps for viewing reports.

Workflow

  1. Connect to data sources
  2. Transform/clean with Power Query
  3. Create data model (relationships)
  4. Write DAX for measures/columns
  5. Build visualizations
  6. Publish to Service
  7. Share dashboards/apps

Data Model Concepts

  • Star Schema: Fact table + dimension tables.
  • Fact Table: Quantitative data (sales, quantities).
  • Dimension Table: Descriptive attributes (customer, product, date).
  • Measures: Aggregations via DAX.
  • Calculated Columns: Row-by-row computed values.
